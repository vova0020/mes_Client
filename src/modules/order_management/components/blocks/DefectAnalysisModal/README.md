# DefectAnalysisModal - Модуль истории обработки

## ⚡ Оптимизация производительности

Модуль полностью оптимизирован для работы с большими объемами данных:
- ✅ Мемоизация всех вычислений и функций
- ✅ Debounce для поиска (300ms)
- ✅ Мемоизированные строки таблиц
- ✅ Ленивая загрузка компонентов
- ✅ CSS-оптимизации для плавного рендеринга

📖 Подробнее: [PERFORMANCE_OPTIMIZATION.md](./PERFORMANCE_OPTIMIZATION.md)

## Структура

Модуль разделен на 5 компонентов:

### 1. DefectAnalysisModal.tsx (Обертка)
- Главный компонент с вкладками
- Управляет переключением между двумя модулями
- Экспортируется по умолчанию

### 2. DefectAnalysis.tsx (Анализ брака)
- Модуль для работы с браком
- Фильтры: период, заказ, материал, станок, этап
- Статистика: общая, по материалам, по станкам, по этапам
- Детальная таблица с информацией о возвратах

### 3. ProductionReport.tsx (Учёт выпуска продукции)
- Модуль для учёта выпуска продукции
- Фильтры: период, рабочее место (станок)
- Поиск с debounce по деталям, материалам, упаковкам
- Статистика: операции, детали, время работы
- Журнал операций с детальной информацией

### 4. DefectTableRow.tsx (Мемоизированная строка таблицы брака)
- Оптимизированный компонент строки таблицы
- Использует `React.memo` для предотвращения лишних рендеров
- Перерисовывается только при изменении данных записи

### 5. ProductionTableRow.tsx (Мемоизированная строка таблицы выпуска)
- Оптимизированный компонент строки таблицы
- Использует `React.memo` для предотвращения лишних рендеров
- Перерисовывается только при изменении данных записи

## Использование

```tsx
import DefectAnalysisModal from './DefectAnalysisModal';

// Использование главного компонента с вкладками
<DefectAnalysisModal onClose={handleClose} />

// Или использование отдельных модулей
import { DefectAnalysis, ProductionReport } from './DefectAnalysisModal';

<DefectAnalysis onClose={handleClose} />
<ProductionReport onClose={handleClose} />
```

## Стили

Каждый компонент имеет свой CSS модуль:
- `DefectAnalysisModal.module.css` - стили обертки с вкладками
- `DefectAnalysis.module.css` - стили модуля анализа брака
- `ProductionReport.module.css` - стили модуля учёта выпуска

## API

Все компоненты используют API из:
`../../../../api/orderManagementApi/defectStatisticsApi`

- `getFilterOptions()` - получение опций для фильтров
- `getDefectStatistics(params)` - получение данных о браке
- `getMachineProduction(params)` - получение данных о выпуске

## Производительность

### Оптимизации React:
- **useMemo** - кэширование статистики и фильтрованных данных
- **useCallback** - стабильные ссылки на функции
- **React.memo** - мемоизация компонентов строк таблиц
- **React.lazy** - ленивая загрузка вкладок

### Оптимизации поиска:
- **Debounce 300ms** - задержка перед фильтрацией
- **Мемоизированная фильтрация** - пересчет только при изменении данных

### CSS-оптимизации:
- **will-change** - подсказка браузеру о будущих изменениях
- **contain** - изоляция рендеринга элементов
- **Оптимизированные transitions** - только необходимые свойства

## Рекомендации

### Для больших объемов данных (>1000 записей):
Рассмотрите добавление виртуализации таблиц с помощью `react-window`:

```typescript
import { FixedSizeList } from 'react-window';
```

### Для очень больших объемов (>5000 записей):
Добавьте пагинацию на стороне сервера или клиента.

### Мониторинг производительности:
Используйте React DevTools Profiler для отслеживания производительности в development режиме.
