# Журнал внутренних рекламаций (Невозвращенные детали из брака)

## Описание
Компонент для отображения и фильтрации невозвращенных отбракованных деталей в производстве.

## Структура компонентов

### InternalReclamationsModal
Основной модальный компонент с анимацией открытия/закрытия.

### InternalReclamations
Главный компонент с таблицей данных, фильтрами и статистикой.

### ReclamationTableRow
Компонент строки таблицы для отображения одной записи о невозвращенной детали.

### SearchableSelect
Переиспользуемый компонент выпадающего списка с поиском.

## API

### Эндпоинты

#### Получение опций фильтров
```
GET /statistics/unreturned-defects/filter-options
```
Возвращает списки заказов и упаковок для фильтрации.

#### Получение данных
```
GET /statistics/unreturned-defects?orderId={id}&packageId={id}
```
Возвращает список невозвращенных деталей с возможностью фильтрации.

## Функциональность

### Фильтры
- **Заказ** - выпадающий список с поиском по номеру партии и названию заказа
- **Упаковка** - выпадающий список с поиском по коду и названию упаковки (фильтруется по выбранному заказу)

### Колонки таблицы
1. **Дата** - дата и время обнаружения брака
2. **Заказ** - номер партии и название заказа
3. **Упаковка** - код и название упаковки
4. **Деталь** - код и название детали
5. **Размер детали** - размеры детали
6. **Материал** - название материала и артикул
7. **Невозвращено** - количество невозвращенных деталей (красный бейдж)
8. **Типы дефектов** - список типов обнаруженных дефектов
9. **Этап / Станок** - название этапа производства и станка

### Статистика
- Всего записей
- Невозвращенных деталей (общее количество)
- Всего отбраковано
- Возвращено
- Уникальных заказов
- Уникальных упаковок

## Использование

```tsx
import InternalReclamationsModal from './components/blocks/InternalReclamationsModal';

// В компоненте
const [isOpen, setIsOpen] = useState(false);

<Button onClick={() => setIsOpen(true)}>
  Открыть журнал рекламаций
</Button>

{isOpen && (
  <InternalReclamationsModal onClose={() => setIsOpen(false)} />
)}
```

## Особенности

1. **Ленивая загрузка** - компонент InternalReclamations загружается динамически для оптимизации
2. **Поиск в фильтрах** - выпадающие списки поддерживают поиск по всем полям
3. **Связанные фильтры** - при выборе заказа список упаковок автоматически фильтруется
4. **Адаптивность** - таблица адаптируется под разные размеры экрана
5. **Анимации** - плавное открытие/закрытие модального окна

## Стили
